##UITableView相关       我们在使用UITableView的分组时，有时会想到只使用分组的HeaderView或者只使用FooterView，这时候就需要把不使用的那一部分取消掉，让其不显示。
      那么如何使tableView分组的headerView或者footerView不显示呢？
      一般情况下，我们首先会想到下面这两个方法，然后给它们的返回值设置为0。 ~~~~{ruby} - (CGFloat)tableView:(UITableView *)tableView heightForHeaderInSection:(NSInteger)section{ return 0; } - (CGFloat)tableView:(UITableView *)tableView heightForFooterInSection:(NSInteger)section{ return 0; } ~~~~       但是，当你设置完，并经过反复的试验后，你就会发现，并没有什么卵用。
      经过多方查阅资料，百度、Google以及向同事请教后发现，此处返回的高度不能直接return 0; 若直接return 0，则会显示一段默认的高度，而这个默认的高度并不是我们想要的。正确的写法是：返回一个灰常灰常小的高度，比如0.1，这样就不会再显示默认的高度了，而是显示的我们肉眼并不能看出来的0.1。
       代码如下：
~~~~{ruby} - (CGFloat)tableView:(UITableView *)tableView heightForHeaderInSection:(NSInteger)section{ return 0.1; } - (CGFloat)tableView:(UITableView *)tableView heightForFooterInSection:(NSInteger)section{ return 0.1; } ~~~~
