程序上架后，我们如何收集错误信息？

      现在有很多第三方框架都提供crash收集处理。而崩溃日志是有经验的程序员每天必备的工作。下面介绍一款比较好用的第三方框架：腾讯Bugly

一、概述

1.去年有非常多的第三方框架做错误收集处理【友盟/腾讯bugly/国外的】

2.每个框架使用都差不多！

3.bugly通过一年的改进，现在使用最方便，一句代码搞定，但是这句代码一定要放在第一句，一旦启动后就调用。

二、腾讯bugly使用过程？

1.登录bugly网站，创建app

图1.创建app

2.使用cocoaPods集成框架

pod 'Bugly'

3.导入头文件 

#import <Bugly/Bugly.h>
4.在工程AppDelegate.m的application:didFinishLaunchingWithOptions:方法中初始化Bugly

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions { 
        [Bugly startWithAppId:@"此处替换为你的AppId"]; 
        return YES; 
  }
